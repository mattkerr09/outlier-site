Ternary MoE Runtime

Run 405B-class models on your own hardware.

Outlier is a patented ternary-quantized Mixture-of-Experts architecture and runtime that runs 72B–405B+ models on 40–65 GB RAM using {-1,0,1} experts, an INT4 shared backbone, and NVMe expert paging.

Patent pending

Current deployment target

Outlier is focused on sovereign inference for operators who need 72B–405B+ class models on hardware they can own, audit, and deploy locally.

The stack combines ternary experts, a higher-precision shared backbone, and storage-aware runtime design so large models stay usable on a Mac Studio, a single GPU node, or future edge-class systems.

Models shipped

Outlier-7B-v0, Outlier-72B-v0, Outlier-405B-v0 on HuggingFace (Outlier-AI org).

Memory footprint

72B = 37–40 GB; 405B = 50–65 GB with TurboQuant KV + NVMe paging.

7B MMLU baseline

28.5% → 50.0% from ternary shadow injection (pre-SFT).

Hardware target

Mac Studio 64 GB, single high-end GPU node, phone-scale 3B in 1.5 GB (roadmap).

Architecture

Ternary MoE with a higher-precision anchor.

The architecture keeps dense-model priors where they matter, compresses expert capacity aggressively, and moves the rest of the problem into routing and runtime discipline rather than cloud-scale memory budgets.

Shared Backbone

Pretrained dense model retained as shared INT4 expert.

Shared INT4 expert from pretrained dense model (Qwen2.5-7B, Llama-3-405B).

Sparse Experts

Eight ternary experts per layer.

8 ternary experts per layer with weights in {-1,0,1}.

Routing

Per-token activation stays bounded.

Router activates 1–2 experts per token, compute ~8–20B active params per token while total params ~168B (72B) and ~900B (405B).

Quality Anchor

The shared expert never drops out.

Shared expert always active as a higher-precision quality anchor.

Runtime

Storage-aware inference pipeline.

NVMe expert paging + TurboQuant 3.5-bit KV cache → 32K context in 40–65 GB RAM.

System Result

Large total parameter count, consumer-class residency.

The point is not just compression. It is making frontier-class model capacity practical on single-device and small-node deployments without abandoning useful scale.

Current Status

Models are live. Training work is active.

The current state is not conceptual. Base models are published, larger-weight handling has been proven in practice, and the next training cycle is aimed at measuring post-SFT quality lift.

  • Outlier-7B-v0, Outlier-72B-v0, Outlier-405B-v0 live on HuggingFace (Outlier-AI).
  • 405B uploaded as 191 shards using 2×A100 80 GB.
  • 7B PTQ baseline 28.5% MMLU → 50.0% after shadow injection.
  • 7B QLoRA knowledge-distillation fine-tune currently running on 2×A100 80 GB to measure post-SFT MMLU.
Research Direction

Path to higher-quality 405B-class deployment

Deep research shows path to 88–92 MMLU at 405B using CAKLD, LoTA-QAF, TurboQuant KV, TriSpec, Sparse-BitNet 2:4.

Operational Detail

The immediate work is validating the 7B training lift after supervised fine-tuning, then using those results to guide 72B and 405B v1 and v2 runs.
