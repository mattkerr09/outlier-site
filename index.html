Architecture

Ternary experts, full-precision anchor.

Each layer keeps the pretrained dense FFN as a shared full-precision expert and adds domain-specialized ternary {-1, 0, +1} routed experts. Top-2 routing means only a fraction of total parameters activate per token.

△

Ternary weights

Expert weights constrained to {-1, 0, +1} with learned per-group scaling. ~1.85 bits per parameter. Multiplication-free inference via lookup tables.

⇄

Sparse MoE routing

Top-2 expert activation per token. 155B total params but only ~75B active per forward pass. Compute scales with active params, not total params.

■

Shared expert anchor

Original pretrained FFN frozen as a shared expert. Always active. Prevents quality collapse that pure ternary models suffer from.

◇

NVMe expert paging

Hot experts in RAM, cold experts streamed from SSD. Measured 2× throughput advantage for ternary vs INT4 on paged expert loads.

⊞

Cross-platform runtime

MLX (Apple Silicon), CUDA (NVIDIA), and CPU (AVX/NEON) backends. Auto-detection at startup. 84 tests passing.

◎

Expert packs (.oep)

Modular, installable expert modules. Add medical, legal, or code expertise post-deployment without retraining the base model.
