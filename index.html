Architecture

Ternary experts. Frozen base. Tiny overlay.

Each MoE layer adds ternary routed experts on top of an unmodified Qwen2.5 base. The overlay is the product; the base is a dependency you already have. The whole recovery trick for the flagship is a 15 KB file.

△

Ternary experts

Routed expert weights constrained to {-1, 0, +1} with per-expert learned scaling. ~1.6 bits per weight. Stored as an overlay on a frozen Qwen2.5 base, not as a standalone checkpoint.

■

Frozen Qwen base

Pretrained Qwen2.5 base runs unmodified as the shared full-precision anchor. Always active. Provides the quality floor; the ternary experts add specialization and recover capacity via CAKLD distillation.

⇄

Sparse routing

Top-K routing per MoE layer activates a subset of experts per token. Compute scales with active parameters, not total. 68B total / 32B active on the V3.3 flagship.

◇

Paged expert cache

Three-tier hot/warm/cold cache with cross-layer prefetch at 81.5% accuracy. Hot experts in RAM, cold experts stream from disk. Enables the 150B-total model on workstation-class hardware.

✦

Alpha-fix recovery

Novel post-training recovery primitive. Training 280 per-expert scalar gates in 18 minutes recovered +1.61pp MMLU on 68B-A32B where a 68M-parameter LoRA regressed. 15 KB overlay file. Fourth provisional under novelty review.

⊞

Apache 2.0 runtime

Open-source inference engine. MPS, CUDA, and CPU backends. lm-eval compatible. Overlay loader with alpha gating built in. Reproduces every number on this page from a saved source file.

Model Family

Four scales. Base + Instruct.

Each scale ships as two SKUs. Base is the research variant — raw weights, Apache 2.0, no safety training, clearly labeled. Instruct is the default download — same weights plus a Llama Guard 3 wrapper and (on 68B-A32B) a DPO safety adapter. Naming follows the TotalB-AyyB convention used by DeepSeek, Mixtral, and Llama 4.
