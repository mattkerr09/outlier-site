Architecture

Ternary experts. Full-precision anchor.

Each layer keeps the pretrained dense FFN as a shared full-precision expert and adds domain-specialized ternary routed experts. Top-2 routing means only a fraction of total parameters activate per token.

△

Ternary weights

Expert weights constrained to {-1, 0, +1} with learned per-group scaling. ~1.85 bits per parameter. Multiplication-free inference via lookup tables.

⇄

Sparse MoE routing

Top-2 expert activation per token. 36B total but ~14B active per forward pass. Compute scales with active parameters, not total.

■

Shared expert anchor

Original pretrained FFN frozen as shared expert. Always active. Prevents the quality collapse pure ternary models suffer from.

◇

NVMe expert paging

Hot experts in RAM, cold experts stream from SSD. Measured 2× throughput advantage for ternary vs INT4 on paged expert loads.

⊞

Cross-platform runtime

Open-source inference engine. CPU backend working, Metal/MLX backend in development. Apache 2.0 licensed.

◎

Expert packs

Modular .oep format for installable expert modules. Add domain expertise post-deployment without retraining the base model.
